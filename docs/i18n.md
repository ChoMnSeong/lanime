# Lanime i18n 가이드

## 전체 구조

Lanime의 i18n은 두 가지 방식으로 처리됩니다.

| 방식 | 대상 | 처리 주체 |
|------|------|-----------|
| **번역 테이블** | `animation.title`, `animation.description`, `episode.title`, `episode.description` | 서버 |
| **키 기반** | `status`, `airDay`, `genre`, `animationType` 등 카테고리 값 | 클라이언트 (i18next) |

---

## 1. 번역 테이블 방식 (서버 처리)

### 동작 원리

클라이언트가 `Accept-Language` 헤더를 전송하면 서버가 해당 언어로 번역된 콘텐츠를 반환합니다.

```
GET /api/v1/animations
Accept-Language: ko

→ title, description이 한국어로 반환
```

### Fallback 우선순위

```
요청 locale (예: ko) → ja (기본) → 원본 컬럼값
```

번역이 없는 경우 `ja` → DB 원본 순으로 자동 fallback합니다.

### 지원 대상 API

| 엔드포인트 | 번역 필드 |
|-----------|----------|
| `GET /api/v1/animations` | `title`, `description` |
| `GET /api/v1/animations/{id}` | `title`, `description` |
| `GET /api/v1/animations/{id}/episodes` | `title`, `description` |
| `GET /api/v1/animations/weekly` | `title`, `description` |
| `GET /api/v1/animations/rankings` | `title` |
| `GET /api/v1/watch-history` | episode `title`, animation `title` |

### DB 스키마

```sql
animation_translation (
    animation_id UUID  -- FK → animation
    locale       VARCHAR(10)   -- 'ko', 'en', 'ja'
    title        TEXT
    description  TEXT
    PRIMARY KEY (animation_id, locale)
)

episode_translation (
    episode_id UUID  -- FK → episode
    locale     VARCHAR(10)
    title      TEXT
    description TEXT
    PRIMARY KEY (episode_id, locale)
)
```

### 번역 등록 API (어드민)

```
PUT /api/v1/admin/animations/{animationId}/translations/{locale}
Authorization: Bearer {adminToken}

{
  "title": "귀멸의 칼날",
  "description": "대마살상부대..."
}
```

```
PUT /api/v1/admin/episodes/{episodeId}/translations/{locale}
Authorization: Bearer {adminToken}

{
  "title": "최종 결전",
  "description": "..."
}
```

- locale 예시: `ko`, `en`, `ja`, `zh`
- 이미 번역이 있으면 덮어씀 (upsert)
- 애니메이션 생성(`POST /api/v1/admin/animations`) 시 입력한 title/description은 자동으로 `ja`로 저장됨

## 2. 키 기반 방식 (클라이언트 처리)

카테고리성 값은 서버가 영문 키를 반환하고, 클라이언트가 i18next로 변환합니다.

### 클라이언트 적용 방법

```js
t(`animationStatus.${animation.status}`)   // "방영 중"
t(`airDay.${animation.airDay}`)             // "금요일"
t(`genre.${genre}`)                         // "백합"
t(`rankingType.${type}`)                    // "실시간"
```

### AnimationStatus

| 키 | 한국어 | 영어 |
|----|--------|------|
| `UPCOMING` | 방영 예정 | Upcoming |
| `ONGOING` | 방영 중 | Ongoing |
| `FINISHED` | 완결 | Finished |

### AirDay

| 키 | 한국어 | 영어 |
|----|--------|------|
| `MONDAY` | 월요일 | Monday |
| `TUESDAY` | 화요일 | Tuesday |
| `WEDNESDAY` | 수요일 | Wednesday |
| `THURSDAY` | 목요일 | Thursday |
| `FRIDAY` | 금요일 | Friday |
| `SATURDAY` | 토요일 | Saturday |
| `SUNDAY` | 일요일 | Sunday |

### RankingType

| 키 | 한국어 | 영어 |
|----|--------|------|
| `REALTIME` | 실시간 | Realtime |
| `Q1` | 1분기 | Q1 |
| `Q2` | 2분기 | Q2 |
| `Q3` | 3분기 | Q3 |
| `Q4` | 4분기 | Q4 |
| `LAST_YEAR` | 전년도 | Last Year |
| `ALL` | 역대 | All Time |

### EncodingStatus

| 키 | 한국어 | 영어 |
|----|--------|------|
| `PENDING` | 인코딩 대기 | Pending |
| `ENCODING` | 인코딩 중 | Encoding |
| `COMPLETED` | 인코딩 완료 | Completed |
| `FAILED` | 인코딩 실패 | Failed |

### Genre (DB 관리 — 관리자가 추가/삭제)

| 키 | 한국어 | 영어 |
|----|--------|------|
| `YURI` | 백합 | Yuri |

> 관리자가 장르를 추가할 경우 반드시 **영문 대문자 키**로 등록하고,
> 클라이언트 i18n 파일에 해당 키의 번역을 추가해야 합니다.

### AnimationType (DB 관리 — 관리자가 추가/삭제)

| 키 | 한국어 | 영어 |
|----|--------|------|
| `TVA` | TVA | TVA |
| `OVA` | OVA | OVA |
| `ONA` | ONA | ONA |

> 애니메이션 타입은 고유명사이므로 번역 없이 키 그대로 표시합니다.

## 3. 신규 콘텐츠 등록 워크플로우

### 애니메이션 등록

```
1. POST /api/v1/admin/animations
   - title/description은 ja(원본)로 자동 저장됨

2. PUT /api/v1/admin/animations/{id}/translations/ko
   { "title": "귀멸의 칼날", "description": "..." }

3. PUT /api/v1/admin/animations/{id}/translations/en
   { "title": "Demon Slayer", "description": "..." }
```

### 에피소드 등록

```
1. POST /api/v1/admin/animations/{id}/episodes
   - title은 ja(원본)로 자동 저장됨

2. PUT /api/v1/admin/episodes/{id}/translations/ko
   { "title": "최종 결전" }
```

### 신규 장르 추가

```
1. POST /api/v1/admin/animations/genres
   { "name": "ACTION" }   ← 영문 대문자 키로 등록

2. 개발팀이 클라이언트 언어 파일에 번역 추가
   ko.json → "genre.ACTION": "액션"
   en.json → "genre.ACTION": "Action"

3. 클라이언트 배포
```
